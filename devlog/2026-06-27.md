# 2026-06-27 开发日志

## 字体大小与分页联动修复

### 问题描述
用户反馈三个问题：
1. 字体大时，没看完就到下一章（分页不准确）
2. 字体小时，有很多空白页（分页不准确）
3. 切换章节、退出重进后字号会变小（不是用户设置的值）

### 根本原因
分析发现三个核心问题：

1. **字体变化后没有重新分页**：`update` 中只注入了字体 CSS，但 JS 的 `total` 变量没有更新，导致页数计算错误。

2. **`onPageFinished` 中顺序错误**：先运行分页 JS（用默认字体计算页数），再注入字体 CSS。字体 CSS 生效后浏览器会 reflow，但 JS 的 `total` 变量是旧值。

3. **没有重新分页 API**：`init()` 只在页面加载和窗口 resize 时触发，字体变化无法触发重新分页。

### 修复方案

**1. 添加 `repaginate()` 函数**（ReaderScreen.kt `buildPaginationJs`）：
```javascript
window.repaginate = function(keepPage) {
    if (!wrap || !outer) return;
    vw = innerWidth; vh = innerHeight;
    wrap.style.columnWidth = vw + 'px';
    var tw = wrap.scrollWidth;
    var newTotal = Math.max(1, Math.ceil(tw / vw));
    if (newTotal !== total) {
        total = newTotal;
        wrap.style.width = (total * vw) + 'px';
    }
    if (keepPage && cur >= total) cur = total - 1;
    if (cur < 0) cur = 0;
    wrap.style.transform = 'translateX(' + (-cur * vw) + 'px)';
    try { AndroidBridge.onPageChanged(cur, total); } catch(e) {}
};
```

**2. 修复 `onPageFinished` 注入顺序**：
- **之前**：分页 JS → 字体 CSS（用错误字体计算页数）
- **之后**：字体 CSS → 分页 JS（用正确字体计算页数）

**3. 字体变化时触发重新分页**：
- 使用 `remember { mutableFloatStateOf(fontSize) }` 追踪上一次字体大小
- 检测字体变化后，先注入 CSS，再调用 `window.repaginate(true)`

### 关键代码修改

- `buildPaginationJs()`: 新增 `window.repaginate(keepPage)` 函数
- `onPageFinished`: 调整为先注入字体 CSS，回调中再运行分页 JS
- `HtmlContent` 的 `update` 块：检测字体变化，变化后调用 `repaginate(true)` 重新分页

### 预期效果
- 字体大小变化后，页数会自动重新计算
- 切换章节时，会用正确的字体大小计算分页
- 不会出现空白页或提前翻章的问题
