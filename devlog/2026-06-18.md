### 9. 彻底重构：JS驱动分页和手势

4. **动画**：JS触发后Kotlin播放`Animatable`滑动动画
5. **章节导航**：JS检测到第一页往前/最后一页往后时调用`onChapterFlip`，Kotlin加载新章节HTML

**简化**:
- `BookParser`接口移除`getPageContent`/`getPageCount`/`renderPage`（分页不再由解析器处理）
- `ReaderViewModel`移除`nextPage()`/`previousPage()`，改为`nextChapter()`/`previousChapter()`
- 新增`onPageChanged(page, total)`由JS回调更新页码状态
- 底部显示页码指示器（`currentPage / totalPages`）

### 12. 翻页动画视差效果修正
**文件**: `ReaderScreen.kt`
**问题**:
1. 翻页动画生硬，`ease-out` 缓动不够非线性
2. 两页在同一层级平移，缺少"近大远小"的视差感和层级感
3. 翻页方向与手指方向相反（`offset = -currentPage*vw - dragX` 逻辑错误）

**修正**:

**双层结构（视差）**:
- **底层（body）**：CSS column 分页长条，移动速度 = 手指的 25%，小范围渐显
- **顶层（overlay div）**：拖拽开始时用 `innerHTML` 快照当前页，移动速度 = 100% 跟随手指
- overlay 带 `box-shadow: 0 0 24px rgba(0,0,0,0.18)` 模糊阴影区分层级
- overlay 带 `border-radius: 0 Rpx Rpx 0` 圆角（匹配屏幕 R 角）

**方向修正**:
- 旧代码 `offset = -currentPage * vw - dragX`（方向反了）
- 新代码：overlay 直接 `translateX(dragX)`，body 用 `translateX(-currentPage*vw + dragX*0.25)`
- 手指左滑 → overlay 左移（负值）+ body 左移（负值），方向一致 ✓

**缓动升级**:
- `ease-out` → `cubic-bezier(0.2, 0.8, 0.2, 1)`（快速启动 + 长尾减速，Apple Books 风格）
- 回弹用 `cubic-bezier(0.25, 0.9, 0.25, 1)`（更柔和的弹性）

**参数调整**:
- 翻页距离阈值：30% → 25%（更容易触发）
- 速度阈值：500px/s → 400px/s
- 动画时长：300ms → 350ms（更从容）
- 橡皮筋衰减：0.3 → 0.25（更强的边界感）
- 阴影不透明度：最大 0.18 → 0.55（更明显的层级感）

## 第三轮修复（动画重写 + 多项 bugfix）

### 修复1：文字与屏幕边距过小（PAD_H 8% → 12%）
- `PAD_H = Math.round(vw * 0.12)` — 在 360px 手机上每侧约 43px
- 符合 style.md 的高留白设计原则

### 修复2：文字溢出屏幕（margin 测量缺失）
- **根因**：`paginate()` 用 `el.offsetHeight` 测量高度，但 offsetHeight 不包含 margin
- `<p>` 的 `margin: 1em 0` 等未被计入，导致累积高度超出页面 → 文字溢出
- **修复**：
  ```js
  var cs = getComputedStyle(el);
  var mTop = parseFloat(cs.marginTop) || 0;
  var mBot = parseFloat(cs.marginBottom) || 0;
  var h = el.getBoundingClientRect().height + mTop + mBot;
  ```
- `renderPage()` 中给元素设置 `margin: 0`，防止 margin collapse 导致布局变化
- `unrenderPage()` 中恢复 `margin = ''`

### 修复3：章节末尾空白页 + 无响应
- **根因**：margin 测量缺失导致最后一页可能是 0 高度元素（空白页）
- **修复**：
  1. margin 测量修复后空白页大概率消除
  2. `paginate()` 后过滤空页：`pages = pages.filter(function(p) { return p.length > 0; })`
  3. `window.__paginationReady` 标志，`flipToLastPage()` 检查此标志，未就绪时 setTimeout 重试
  4. `initPagination()` 在分页完成后设置 `__paginationReady = true`

### 修复4：翻页动画重写（Apple Books 双层视差）
- **旧方案**：两层以相同速度 100% translateX 滑动
- **新方案**：双层视差结构
  - 上层（当前页）：100% 跟手，带 `box-shadow` + `border-radius: 0 24px 24px 0`
  - 下层（新页）：25% 速度渐显滑入
  - 边缘阴影：80px 宽渐变条 `rgba(0,0,0,0.10) → transparent`
- **动画参数**：
  - `cubic-bezier(0.2, 0.8, 0.2, 1)` 缓动（快速启动 + 长尾减速）
  - 350ms 翻页 / 300ms 回弹
  - 阴影和圆角在回弹时渐隐到 0
- **Compose 层清理**：
  - 移除 `dragOffsetPx` 状态（阴影/圆角改由 JS CSS 处理）
  - 移除 `handleDragUpdate` 回调
  - 移除 `graphicsLayer` 中的 drag shadow 逻辑
  - 仅保留 `flipOffset` 的 `translationX` 用于跨章节动画

### 改动文件
- `app/src/main/java/com/ebook/reader/ui/reader/ReaderScreen.kt`
  - `buildPaginationJs()` 完全重写（4项修复）
  - `ReaderJsBridge` 移除 `handleDragUpdate` 参数和 `onDragUpdate` 方法
  - `ReaderScreen` 移除 `dragOffsetPx` 状态和相关 graphicsLayer 逻辑

### 修复5：章节末尾翻页后触摸永久锁定
- **根因**：`handleChapterFlipReady` 中 `snapshotFlow` 没有超时保护
  - 如果 `pageReady` 不触发（WebView 加载失败/延迟），`snapshotFlow` 永久挂起
  - `__animating` 永远是 `true`，所有触摸事件被阻塞
- **修复**（彻底方案，非超时兜底）：
  1. **JS → Kotlin 回调**：`initPagination()` 完成后调用 `AndroidBridge.onPaginationComplete()`
  2. **CompletableDeferred**：Compose 端用 `CompletableDeferred<Unit>` 等待信号，替代被动的 `snapshotFlow`
  3. **5s 超时**：`withTimeoutOrNull` 作安全网（JS 异常时兜底）
  4. **try-finally**：无论成功还是超时，`finally` 中重置 `__animating = false`
- **效果**：章节切换由 JS 主动通知完成，不再被动轮询状态；即使异常也有超时 + finally 双保险

### 修复6：上下边距不足（避开状态栏/摄像头）
- `PAD_V`: 16px → **54px**（约 1.9cm）
- 上下各留 54px，确保内容不被状态栏、刘海、摄像头遮挡

### 修复7：松手翻页动画从头播放
- **根因**：`flipTo()` 用 `startTopX = 0` 作为动画起点，不管当前拖拽位置
- **修复**：新增 `getCurrentX(el)` 函数，从 `style.transform` 提取当前 `translateX`
  - `flipTo()` 中 `startTopX = getCurrentX(activeSlot)` — 从松手位置继续
- **效果**：拖拽松手后，页面从当前位置平滑完成翻页，不会跳回起点

### 修复8：动画速率优化
- 缓动函数：`cubic-bezier(0.2, 0.8, 0.2, 1)` → **`1 - Math.pow(1 - t, 3)`**（ease-out cubic）
  - 快启动 + 长尾缓减速，更接近 Apple Books 的"吸附"感
- 回弹缓动：`1 - Math.pow(1 - t, 4)`（ease-out quart，更柔和的弹性）
- 时长：翻页 350ms → **450ms**，回弹 300ms → **400ms**
- 跨章节动画：300ms → **350ms**

### 修复9：下层页面多余阴影
- **根因**：上层（activeSlot）有 `box-shadow` 正确；下层（inactive）也被设了 `box-shadow`（虽然是 none，但有 borderRadius 残留）
- **修复**：下层 slot 始终 `boxShadow = 'none'` + `borderRadius = '0'`，确保干净无阴影
- 移除了边缘阴影元素（`edgeShadow`）——简化视觉，只保留上层阴影

### 修复10：分页测量 + 文字级精度拆分
- **根因**：旧方案修改元素 `margin:0` 导致测量/渲染布局不一致；段落级切分导致一页一段
- **新方案**：文字级精度分页
  1. 测量 slot（`height:auto`）内逐个添加元素，`scrollHeight` 实测
  2. 不修改元素 margin，保持原生样式
  3. 元素放不下时 → 二分查找文本断点（`findSplitIndex`）
     - 用 `<p>` + `copyStyles` 创建测量元素
     - 二分查找 `text.substring(0, mid)` 的高度 ≤ 剩余空间
     - 精度：字符级
  4. 创建两个新元素：`fitP`（当前页）+ `remP`（下一页）
  5. 复制原元素的所有文字样式（font/lineHeight/letterSpacing/textAlign 等）
- **关键函数**：
  - `findSplitIndex(text, cs, maxH, measureEl, ms)` — 二分查找断点
  - `createSplitEl(text, cs, tag)` — 创建拆分元素
  - `copyStyles(target, cs)` — 复制文字样式
